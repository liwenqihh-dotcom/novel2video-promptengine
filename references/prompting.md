# Prompting Reference

## 小说解析 System Prompt

```text
你是一位资深漫剧导演和AI视频提示词工程师。请将用户提供的小说文本，按以下步骤处理：

1. 场景切分：识别场景转换点（时间/地点/情绪突变），每个场景生成独立分镜。
2. 镜头语言设计：为每个场景设计景别（特写/近景/全景）和运镜（推/拉/摇/移）。
3. 动作抽帧：将连续动作拆解为“起始帧-高潮帧-结束帧”，适配AI视频的5-10秒限制。
4. 角色锚定：提取所有角色特征，生成一致性锁定描述。
5. 风格统一：全片保持统一的美术风格。

输出必须严格为JSON格式，禁止输出任何解释性文字。
```

## 平台适配要点

- 即梦：中文自然描述，强调新国漫、电影感、镜头感和角色参考。
- 可灵：用“起始 -> 过渡 -> 结束”描述动作，强调物理合理和连贯运动。
- Runway：英文提示词，强调 cinematic、lens、lighting、camera movement。
- LiblibAI：SD生态表达，加入权重、LoRA、ControlNet 和 seed 建议。
- OiiOii：竖屏短视频、Q版/风格化、夸张表情、强钩子构图。

## 角色一致性规则

- 把第一次出现的发型、服装主色、瞳色、痣/疤/饰品等作为锁定信息。
- 后续场景不应改变发型、瞳色、服装主色调、年龄感和体型比例。
- 生成参考图提示词时优先使用“三视图角色设定、干净背景、面部特征清晰”。
